(function () {
    'use strict';
    var _sessionCache = {};
    function clearSession() { Object.keys(_sessionCache).forEach(function(k){ delete _sessionCache[k]; }); }
    function fetchForTicker(ticker, lookback) {

  <action>
    Create static/js/mlSignals.js as a complete IIFE. Implement in this order:

    1. IIFE wrapper: `(function () { 'use strict'; ... window.MLSignals = { fetchForTicker, clearSession, fetchPCA }; })();`

    2. Session cache: `var _sessionCache = {};` keyed by ticker string (no lookback — ML signals have no lookback dropdown).

    3. clearSession(): delete all keys from _sessionCache (same pattern as tradingIndicators.js).

    4. _createTickerShell(ticker): find or create div#ml-card-{ticker} inside #mlSignalsTabContent; set innerHTML to loading skeleton `<div class="ml-ticker-header"><h3>{ticker}</h3></div><p class="ml-loading" style="color:#7f849c;">Training ML models — this may take a few seconds...</p>`.

    5. fetchForTicker(ticker):
       - Check _sessionCache[ticker]; if hit, return early.
       - Set _sessionCache[ticker] = true.
       - Call _createTickerShell(ticker).
       - Fire 4 parallel fetches using Promise.all for features: direction, regime, credit, lstm.
       - On resolve: call _renderTickerCard(ticker, dirData, regData, credData, lstmData).
       - On reject: call _renderError(ticker, err.toString()).

    6. fetchPCA(tickers):
       - Build URL: '/api/ml_signals?feature=pca&' + tickers.map(function(t){ return 'tickers=' + encodeURIComponent(t); }).join('&')
       - Find or create div#mlSignalsPcaSection inside #mlSignalsTabContent (insert before first ticker card).
       - On success: if result.pca_available === false, show "Add more tickers to enable PCA decomposition" in grey text. Else render scree plot (result.scree_traces + result.layout) and heatmap (result.heatmap_traces + result.layout) in two Plotly divs with ids ml-pca-scree and ml-pca-heatmap.

    7. _renderTickerCard(ticker, dirData, regData, credData, lstmData):
       Build HTML string and set div#ml-card-{ticker}.innerHTML, then call Plotly for chart divs:

       Section A — Direction Signal (RF):
       - If dirData.insufficient_data: grey card "Insufficient history (< 265 trading days) — cannot train direction signal."
       - Else: badge span with text "{dirData.confidence*100|round}% {dirData.signal}" coloured #2ecc71 (Bullish) or #e74c3c (Bearish). Create div id="ml-dir-chart-{ticker}" for Plotly. After setting innerHTML, call: `if (dirData.traces && dirData.traces.length) Plotly.newPlot('ml-dir-chart-' + ticker, dirData.traces, dirData.layout || {}, { staticPlot: true, responsive: true });`

       Section B — K-Means Regime:
       - Two inline badges: "HMM: {regData.hmm_regime || 'N/A'}" and "K-Means: {regData.current_regime}".
       - If regData.models_agree === true: small green text "Models agree". If false: small amber text "Models diverge".
       - Create div id="ml-reg-chart-{ticker}" for timeline. After innerHTML: `if (regData.regime_timeline_traces && regData.regime_timeline_traces.length) Plotly.newPlot('ml-reg-chart-' + ticker, regData.regime_timeline_traces, regData.layout || {}, { staticPlot: true, responsive: true });`

       Section C — Credit Risk Score:
       - If credData.degenerate_labels or credData.insufficient_data: grey card "Credit model could not train (insufficient label variation for this ticker)."
       - Else: compute distressPct = Math.round((credData.p_distress || 0) * 100). Colour: <30% green, 30-60% #f39c12 amber, >60% #e74c3c red. Show "{distressPct}% probability of financial distress". List top 3 factors from credData.top_factors as bullet points. Show credData.caveat in small grey italic.

       Section D — LSTM Direction Signal:
       - If lstmData.lstm_available === false: grey card "Deep learning disabled on cloud — run locally to enable."
       - Else: badge + confidence + loss curve chart in div id="ml-lstm-chart-{ticker}". If dirData.signal and lstmData.signal: show comparison row — same signal → green "RF and LSTM agree: {signal}", different → amber "RF: {dirData.signal} | LSTM: {lstmData.signal} — signals diverge".
       - After innerHTML: `if (lstmData.loss_curve_traces && lstmData.loss_curve_traces.length) Plotly.newPlot('ml-lstm-chart-' + ticker, lstmData.loss_curve_traces, lstmData.layout || {}, { staticPlot: true, responsive: true });`

    8. _renderError(ticker, message): find div#ml-card-{ticker}; set innerHTML to `<p style="color:#e74c3c;">Error loading ML signals: {message}</p>`.

    Use `var` throughout (not `const`/`let`) to match existing project JS convention in tradingIndicators.js (which uses var).
    All Plotly.newPlot calls wrapped in `if (typeof Plotly !== 'undefined')` guard.
