```javascript
async sendEmail(emailData) {
    const controller = new AbortController();
    const timeoutId = setTimeout(() => controller.abort(), 30000);
    try {
        const response = await fetch('/api/send-email', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(emailData),
            signal: controller.signal
        });
        clearTimeout(timeoutId);
        if (!response.ok) {
            const errorText = await response.text();
            throw new Error(`Failed to send email (${response.status}): ${errorText}`);
        }
        return await response.json();
    } catch (error) {
        if (error.name === 'AbortError') throw new Error('Email request timed out. Please try again.');
        throw error;
    }
},
```

    ```javascript
    /**
     * Export stock data to Google Sheets
     * @param {Object} exportData - {tickers, data, trading_indicators_data}
     * @returns {Promise<Object>} - {success, rows_added, ti_rows_added?} or {success, rows_added, warning?}
     */
    async exportSheets(exportData) {
        const controller = new AbortController();
        const timeoutId = setTimeout(() => controller.abort(), 30000);

        try {
            const response = await fetch('/api/export-sheets', {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/json'
                },
                body: JSON.stringify(exportData),
                signal: controller.signal
            });

            clearTimeout(timeoutId);

            if (!response.ok) {
                const errorText = await response.text();
                throw new Error(`Export failed (${response.status}): ${errorText}`);
            }

            const contentType = response.headers.get('content-type');
            if (!contentType || !contentType.includes('application/json')) {
                throw new Error('Invalid response format from export service');
            }

            return await response.json();
        } catch (error) {
            if (error.name === 'AbortError') {
                throw new Error('Export timed out. Please try again.');
            }
            throw error;
        }
    },
    ```

    The alert logic implements CONTEXT.md 2026-05-21 locked formats:
    - Full success (result.ti_rows_added is a number): show success type with TI count
    - Partial success (result.warning): show warning type with the warning string
    - No TI data exported (neither ti_rows_added nor warning in result): show success without TI mention

    ```javascript
    /**
     * Export stock data to Google Sheets (named tabs + Trading Indicators tab).
     * Alert formats per CONTEXT.md 2026-05-21:
     *   Full success:    "Exported N tickers to Google Sheets ✓ (Trading Indicators: M tickers)"  [success]
     *   Partial success: "Exported N tickers to Google Sheets ✓ — Warning: Trading Indicators: X" [warning]
     *   Failure:         "Google Sheets export failed: <reason>"                                   [error]
     */
    async exportSheets() {
        if (!AppState.currentData) {
            Utils.showAlert('No data to export. Please analyze stocks first.', 'error');
            return;
        }

        const btn = document.getElementById('exportSheetsBtn');

        try {
            if (btn) btn.disabled = true;
            Utils.showAlert('Exporting to Google Sheets...', 'info');

            const exportData = {
                tickers: AppState.currentTickers,
                data: AppState.currentData,
                trading_indicators_data: AppState.tradingIndicatorsData || {}
            };

            const result = await API.exportSheets(exportData);

            if (result.success) {
                const rowsMsg = `Exported ${result.rows_added} tickers to Google Sheets ✓`;

                if (result.warning) {
                    // Partial success — TI tab failed
                    Utils.showAlert(`${rowsMsg} — Warning: ${result.warning}`, 'warning');
                } else if (typeof result.ti_rows_added === 'number') {
                    // Full success with TI data
                    Utils.showAlert(`${rowsMsg} (Trading Indicators: ${result.ti_rows_added} tickers)`, 'success');
                } else {
                    // Full success, no TI data was sent
                    Utils.showAlert(rowsMsg, 'success');
                }
            } else {
                Utils.showAlert('Google Sheets export failed: ' + result.error, 'error');
            }
        } catch (error) {
            Utils.showAlert('Google Sheets export failed: ' + error.message, 'error');
        } finally {
            // Re-enable only if sheets credentials are configured.
            // data-sheets-unconfigured="true" means server-rendered disabled due to missing creds.
            if (btn && !btn.dataset.sheetsUnconfigured) {
                btn.disabled = false;
            }
        }
    },
    ```

    ```markdown
    ## Google Sheets Setup

    The "Export to Sheets" feature lets you upsert scraped stock data directly into a Google Sheet you own — one row per ticker, matched by ticker column and updated in-place on re-export. Data is written to named tabs by exchange (`US Stock`, `SG Stock`, `HK Stock`, `Others Stock`). If you have run the Trading Indicators analysis, a `Trading Indicators` tab is auto-created on first export. This is a local-only feature — not available on the Render deployment.

    ### Prerequisites

    - A Google account with access to Google Sheets
    - Python dependencies: `gspread==6.2.1` and `google-auth==2.52.0` (already in `requirements.txt`)

    ### Step-by-Step Setup

    **1. Create a Google Cloud project and enable the Sheets API**
    - Go to [Google Cloud Console](https://console.cloud.google.com/)
    - Create a new project (or use an existing one)
    - Navigate to "APIs & Services" → "Enable APIs and Services"
    - Search for "Google Sheets API" and click Enable

    **2. Create a service account and download credentials**
    - In the Cloud Console, go to "APIs & Services" → "Credentials"
    - Click "Create Credentials" → "Service Account"
    - Give it a name (e.g. `finance-scraper-sheets`)
    - Click "Done"
    - Click on the service account email → "Keys" tab → "Add Key" → "Create new key" → JSON
    - Download the JSON file — note the path (e.g. `/Users/yourname/credentials.json`)

    **3. Share your Google Sheet with the service account**
    - Create or open the target Google Sheet in your browser
    - Add these named tabs (case-sensitive): `US Stock`, `SG Stock`, `HK Stock`, `Others Stock`
    - Click "Share" (top right)
    - Copy the service account email (found in the downloaded JSON under `client_email`)
    - Paste it into the Share dialog and grant **Editor** access
    - Click "Send"

    **4. Configure your `.env` file**
    - Open `.env` (copy from `.env.example` if not created yet)
    - Copy the Spreadsheet ID from your Sheet's URL — it is the long alphanumeric string between `/d/` and `/edit`:
      ```
      https://docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/edit
      Spreadsheet ID: 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms
      ```
    - Add to `.env`:
      ```
      GOOGLE_SHEETS_CREDENTIALS_PATH=/absolute/path/to/your/credentials.json
      GOOGLE_SHEETS_SPREADSHEET_ID=your_spreadsheet_id_here
      ```

    **5. Add header rows to your named tabs (first time only)**
    - In row 1 of each tab, add the column headers in order. The `Trading Indicators` tab is auto-created on first export — no manual setup needed for that tab.
    - Fundamentals headers (24 columns):
      `Export Date | Ticker | Price | P/E | Forward P/E | P/B | EPS | RSI | MA10 Signal | MA20 Signal | MA50 Signal | Sentiment Score | Revenue | Profit Margin | Operating Margin | Debt/Equity | Health Score | Earnings Quality Flag | DCF Intrinsic Value | Peer P/E Percentile | Ticker Summary | Recommended Action | Analysis Methods | Data Source Credibility`

    ### Usage

    1. Start the webapp: `python webapp.py`
    2. Scrape one or more tickers using the Stock Analysis tab
    3. (Optional) Open the Trading Indicators tab for tickers you want to include TI data for
    4. Click **"📊 Export to Sheets"** — the button appears next to the Email Report form
    5. A success message shows how many tickers were exported and how many TI rows were written
    6. Each export upserts rows — existing tickers are updated in-place, new tickers are appended

    ### Troubleshooting

    | Error | Cause | Fix |
    |-------|-------|-----|
    | "Spreadsheet not found or not shared" | Service account not shared with your Sheet | Share the Sheet with the service account email as Editor |
    | "Credentials file not found" | Path in GOOGLE_SHEETS_CREDENTIALS_PATH is wrong | Check the path exists: `ls $GOOGLE_SHEETS_CREDENTIALS_PATH` |
    | "GOOGLE_SHEETS_CREDENTIALS_PATH is not set" | Missing `.env` entry | Add the env var to `.env` |
    | Button shows tooltip "Configure credentials" | Both env vars not set | Complete Step 4 above |
    | Warning: "Trading Indicators: ..." | TI tab export failed (fundamentals wrote successfully) | Check that the service account has Editor access; fundamentals data was saved |
    ```
