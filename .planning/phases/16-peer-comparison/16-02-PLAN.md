  <action>
    Create static/js/peerComparison.js as an IIFE following the exact EarningsQuality module pattern.

    The module must expose: window.PeerComparison = { renderIntoGroup, clearSession }

    Internal functions needed:

    buildLoadingHTML(): returns a non-expandable section header div showing a spinner character. Use the deep-analysis-section CSS class for consistent styling. Include a spinning indicator using inline style (animation:spin not required - a simple rotating character like the circled X or just "..." is fine for the loading state).

    buildFailureHTML(): returns a section div with opacity 0.55 and text "Peer Comparison: Unavailable". No expand arrow. Not interactive.

    buildSuccessHTML(ticker, resp): builds the full success HTML including:
      1. A collapsible header showing "N/4 above median" count (click to expand/collapse content)
      2. Content area (hidden by default) containing:
         a. Four metric rows: P/E Ratio, P/B Ratio, ROE, Op. Margin
            Each row: metric name | "{rank}th percentile" | badge (badge-success if rank >= 50, badge-danger otherwise)
            Null rank values: display "-- percentile" and badge-secondary or no badge
         b. Peer group label line: "Comparable group: {resp.peers.join(', ')}"
         c. A button with class "peer-toggle-btn" and text "Show peers"
         d. A table with class "peer-raw-table" style "display:none"
            Columns: Ticker | P/E | P/B | ROE | Op. Margin
            Rows: one per entry in resp.peer_data; null values shown as single dash

    _wireToggle(sectionEl): attaches click handler to .peer-toggle-btn to show/hide .peer-raw-table.
    Wire header click to collapse/expand content area (check healthScore.js for the onclick pattern on the deep-analysis-header element and replicate it identically).

    _fetchAndRender(ticker, sectionEl):
      Uses fetch('/api/peers?ticker=' + encodeURIComponent(ticker)).then(r => r.json())
      On error response or fewer than 2 peer_data entries: replace sectionEl.innerHTML with buildFailureHTML()
      On success: replace sectionEl.innerHTML with buildSuccessHTML(ticker, resp); call _wireToggle(sectionEl); write to pageContext

    renderIntoGroup(ticker, data, cardRoot):
      Guard against double-render using _sessionCache[ticker]
      Queries cardRoot for #deep-analysis-content-{ticker}
      Creates section element, injects buildLoadingHTML(), appends to container
      Calls _fetchAndRender(ticker, section) fire-and-forget (no await)

    clearSession(): deletes all keys from _sessionCache

    Expose as window.PeerComparison and also module.exports for test environments.
