# Plan 10.1-01 Summary: FinancialAnalyst Backend Persona

**One-liner:** Added module-level `SYSTEM_PROMPTS` dict and `agent` field dispatch to `/api/chat`, validated by 3 TDD tests.

## What Was Built

### SYSTEM_PROMPTS dictionary (webapp.py)

Module-level dict with two entries:
- `'quant'` — existing QuantAssistant prompt (preserves backward compat as default)
- `'financial'` — sell-side FinancialAnalyst persona focused on fundamentals (P/E, EPS, revenue trends)

### Agent dispatch in /api/chat (webapp.py)

```python
agent = data.get('agent', 'quant')
system_prompt = SYSTEM_PROMPTS.get(agent, SYSTEM_PROMPTS['quant'])
```

Reads optional `agent` field from request JSON; defaults to `quant`; unknown keys fall back to `quant` silently.

### Test suite (tests/test_chat_route.py)

Three pytest tests, all passing:
- `test_chat_financial_agent` — agent='financial' → 200 + reply
- `test_chat_default_agent_backward_compat` — no agent field → 200 + reply
- `test_chat_unknown_agent_fallback` — agent='unknown_xyz' → 200 + reply (fallback)

Mocks `webapp.requests.post`; exercises Groq branch via monkeypatched `GROQ_API_KEY`.
