
2. Add `pytest-playwright`, `pytest-flask`, and `responses` to `requirements.txt` (append to the end, under a `# Testing` comment section).

3. Create `Makefile` at project root with these exact targets (per CONTEXT.md locked decision):
   ```makefile
   .PHONY: test test-unit test-integration test-regression test-e2e

   test: test-unit test-integration test-regression test-e2e

   test-unit:
   	pytest -m unit -q

   test-integration:
   	pytest -m integration -q

   test-regression:
   	pytest -m regression -q

   test-e2e:
   	pytest -m e2e -q --browser chromium
   ```

4. Extend `tests/conftest.py` — keep ALL existing content. Add:
   - Register markers in `pytest_configure` (unit, integration, regression, e2e) alongside the existing `slow` marker
   - A `client` fixture (function-scoped) that creates a Flask test client via `webapp.app.test_client()` — import webapp inside the fixture body (not at module top) to avoid heavy import at collection time
   - A `flask_server` fixture (session-scoped) that starts Flask on a random port in a daemon thread for E2E tests — use `socket.bind(('localhost', 0))` to find a free port, `threading.Thread(target=webapp.app.run, kwargs=dict(host='localhost', port=port, use_reloader=False), daemon=True)`, and yield the base URL `f'http://localhost:{port}'`. Include a 0.5s sleep after thread start to let the server bind.
   - Document critical user flows in the module docstring at the top of conftest.py (TEST-01): stock scraping pipeline, stochastic model computation, trading indicator generation, chatbot interaction, portfolio health scoring.

NOTE: Do NOT use pytest-flask's live_server fixture — there is a known teardown hang with playwright (GitHub issue microsoft/playwright-pytest#187). Use the threading pattern instead.
