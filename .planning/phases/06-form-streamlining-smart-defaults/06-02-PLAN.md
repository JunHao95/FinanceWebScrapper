Replace the entire comment + two rules with the following expanded block:
```css
        /* Hero ticker input and Run Analysis button */
        .hero-input-group input { font-size: 1.1em; padding: 14px; }
        #runAnalysisBtn {
            width: 100%;
            font-size: 1.15em;
            padding: 16px 32px;
            letter-spacing: 0.02em;
        }
        /* button-group column layout so Run Analysis is full-width above Clear */
        .form-group ~ .button-group,
        #scrapeForm .button-group {
            flex-direction: column;
            align-items: stretch;
        }
        /* De-emphasise Clear as secondary action */
        #scrapeForm .button-group .secondary-btn {
            font-size: 0.95em;
            padding: 10px 24px;
            opacity: 0.85;
        }
```

Rationale (from RESEARCH.md Pitfall 3): Making #runAnalysisBtn width:100% inside a flex-row .button-group pushes Clear to an awkward second line. Switching the form's .button-group to flex-direction:column and align-items:stretch keeps both buttons full-width in a clean vertical stack, with Run Analysis dominant.

The selector `.form-group ~ .button-group, #scrapeForm .button-group` targets only the scrape-form button group and avoids overriding other .button-group instances elsewhere in the page.

Do not change any HTML in this task — CSS only.
