  <name>Task 1: Create portfolioHealth.js module</name>
  <files>static/js/portfolioHealth.js</files>
  <action>
    Create `static/js/portfolioHealth.js` exposing `window.PortfolioHealth = { initCard, updateRegime }`.

    **Constants** (defined at top of file, not reusing autoRun.js vars — separate file scope):
    ```javascript
    const BADGE_RUNNING_STYLE = 'background:#6c757d;color:white;border-radius:10px;padding:2px 8px;font-size:11px;';
    const BADGE_RISK_ON  = 'background:#28a745;color:white;border-radius:10px;padding:2px 8px;font-size:11px;cursor:pointer;';
    const BADGE_RISK_OFF = 'background:#dc3545;color:white;border-radius:10px;padding:2px 8px;font-size:11px;cursor:pointer;';
    const BADGE_FAILED_STYLE = 'background:#adb5bd;color:white;border-radius:10px;padding:2px 8px;font-size:11px;';
    ```

    **Module-level state:**
    ```javascript
    let _regimeMap = {};     // { TICKER: 'RISK_ON' | 'RISK_OFF' | null }
    let _tickerList = [];    // copy of tickers passed to initCard
    ```

    **`initCard(tickers, analyticsData, allocations)`:**
    1. Remove existing `#portfolioHealthCard` if present (re-run guard)
    2. Set `_tickerList = [...tickers]`; reset `_regimeMap = {}`; initialize `_regimeMap[t] = undefined` for each ticker
    3. Extract VaR using the multi/single-ticker branching logic from RESEARCH.md Pattern 3
    4. Build card HTML with:
       - Card container: `id="portfolioHealthCard"`, inline styles matching existing card aesthetic (white background, subtle border, border-radius:8px, padding:16px, margin-bottom:16px, box-shadow)
       - Traffic-light icon span: `id="healthTrafficLight"`, starts as grey circle ⚪
       - Title: "Portfolio Health"
       - Metric row (flexbox, flex-wrap:wrap, gap:16px):
         - VaR chip: `id="healthVarValue"` shows formatted value (e.g., "12.3%") or "—"; `onclick = () => TabManager.switchTab('analytics')`; cursor:pointer
         - Sharpe chip: `id="healthSharpeValue"` shows "Computing..." initially; `onclick = () => TabManager.switchTab('analytics')`; cursor:pointer
         - Regime badges: one per ticker, each `id="healthRegimeBadge_TICKER"` showing "Analyzing..." with BADGE_RUNNING_STYLE; `onclick = () => TabManager.switchTab('autoanalysis')`
       - Summary line: `id="healthSummaryText"` starts empty; updated after all regimes resolve
    5. Insert card: `tabsContainer.insertAdjacentHTML('beforebegin', cardHTML)` where `tabsContainer = document.getElementById('resultsSection').querySelector('.tabs-container')`
    6. Call `fetchSharpe(tickers, allocations)` — async, updates `#healthSharpeValue` when done

    **`fetchSharpe(tickers, allocations)`** (private async function):
    - Compute 2-year window: same pattern as autoRun.js (today minus 2 years)
    - POST to `/api/portfolio_sharpe` with `{ tickers, weights: allocations, start_date, end_date }`
    - On success: set `#healthSharpeValue` to `data.sharpe.toFixed(2)`
    - On any failure: set `#healthSharpeValue` to "—"

    **`updateRegime(ticker, label)`** (exposed in public API):
    - Guard: `if (!document.getElementById('healthRegimeBadge_' + ticker)) return`
    - Update `_regimeMap[ticker] = label` (label is 'RISK_ON', 'RISK_OFF', or null for failed)
    - Update badge: RISK_ON → green BADGE_RISK_ON + "RISK_ON 🟢"; RISK_OFF → red BADGE_RISK_OFF + "RISK_OFF 🔴"; null → BADGE_FAILED_STYLE + "—"
    - Call `_maybeUpdateSummary()` after each update

    **`_maybeUpdateSummary()`** (private):
    - Return early if any ticker in `_tickerList` still has `_regimeMap[t] === undefined` (still pending)
    - Compute traffic-light: all null → skip; offCount = labels filtered to RISK_OFF; 0 offCount → green ✅; all RISK_OFF → red 🔴; mixed → amber 🟡
    - Update `#healthTrafficLight` text to the appropriate emoji
    - Build summary text per RESEARCH.md Pattern 7 and set `#healthSummaryText`

    **Single-ticker handling (HEALTH-03):** The regime badge loop renders only for tickers in the `tickers` array passed to `initCard`. For single-ticker, `tickers.length === 1` so only one regime badge appears. No correlation/PCA entries are added anywhere in this card — the card structure only ever contains VaR, Sharpe, and regime badges.

    **Anti-patterns to avoid:**
    - Do NOT read from `autoRegimeBadge_*` DOM elements — derive regime from `_regimeMap` state
    - Do NOT re-fetch VaR from any backend — extract from `analyticsData` parameter directly
    - Do NOT use `healthRegimeBadge_TICKER` IDs already used by autoRun (use `healthRegimeBadge_` prefix — different from `autoRegimeBadge_`)
