# Phase 30-04 Plan: Sentiment Company Name Query for SGX Tickers

**Goal:** Sentiment analysis returns meaningful results for SGX tickers (e.g. `D05.SI` → "DBS Group Holdings Ltd") instead of all-zero/No-Data.

**Root cause:** `get_news_sentiment()` in `src/sentiment/sentiment_analyzer.py` builds search terms from the raw ticker string and a hardcoded `company_names` dict of US tickers only:
```python
company_names = {"AAPL": "Apple", "MSFT": "Microsoft", ...}  # no SGX tickers
search_terms = [ticker.lower()]  # → ["d05.si"]
# Google News query: "d05.si" → near-zero results
# BBC/NYT/WSJ/NPR RSS: filter for "d05.si" → 0 matches
```

`D05.SI` as a raw search term returns nothing from US/English RSS feeds. The fix is to resolve the company's human-readable name (yfinance `longName` or `shortName`) and include it in the search query.

**Verified:** yfinance returns `longName = "DBS Group Holdings Ltd"` for D05.SI. A Google News search for `"DBS Group"` returns relevant results.

## Tasks

### Task 1 — Add company name resolution in `enhanced_sentiment_scraper.py`

File: `src/scrapers/enhanced_sentiment_scraper.py`

In `_scrape_data(ticker)`, before calling `self.analyzer.get_comprehensive_sentiment_analysis(ticker)`:

```python
# Resolve company name for non-US tickers (improves news/reddit search quality)
company_name = None
if "." in ticker:
    try:
        import yfinance as yf
        info = yf.Ticker(ticker).info
        company_name = info.get("longName") or info.get("shortName")
    except Exception:
        pass

sentiment_data = self.analyzer.get_comprehensive_sentiment_analysis(
    ticker, company_name=company_name
)
```

yfinance is already a project dependency and used elsewhere — no new package required.

Current:

```python
search_terms = [ticker.lower()]
if ticker.upper() in company_names:
    search_terms.append(company_names[ticker.upper()].lower())
```

Change signature to `get_news_sentiment(self, ticker: str, num_articles: int = 10, company_name: str = None)` and update:
```python
search_terms = [ticker.lower()]
# Use provided company_name first, then fall back to hardcoded dict
resolved_name = company_name or company_names.get(ticker.upper())
if resolved_name:
    search_terms.append(resolved_name.lower())
# For SGX tickers, drop the raw ".SI" suffix form — it pollutes the query
if "." in ticker:
    base = ticker.rsplit(".", 1)[0].lower()
    if base not in search_terms:
        search_terms.append(base)
```

For D05.SI this produces `search_terms = ["d05.si", "dbs group holdings ltd", "d05"]` — Google News finds DBS articles.
