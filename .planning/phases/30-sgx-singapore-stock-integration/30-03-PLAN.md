# Phase 30-03 Plan: Financial Health & Growth Data Gaps for SGX Tickers

**Goal:** Financial Health and Growth sub-scores render meaningful values (not 0.0/10) for SGX-listed banking stocks like DBS (`D05.SI`).

**Root causes (two distinct bugs):**

### Bug A — Revenue Growth not mapped in yahoo_scraper
`yahoo_scraper.py` maps `earningsGrowth` → `"Earnings Growth (Yahoo)"` but never maps `revenueGrowth`. For DBS.SI, `earningsGrowth = None` (not provided by yfinance for SGX banks) but `revenueGrowth = 0.032` and `earningsQuarterlyGrowth = 0.011` are available. Because neither field is in the data dict, `_analyze_growth()` finds `metrics_found == 0` → returns `None` → growth score stays at the initial `0` default.

**Fix:** Map `revenueGrowth` and use `earningsQuarterlyGrowth` as fallback for `earningsGrowth` in `yahoo_scraper.py`.

### Bug B — Financial health score defaults to 0.0 instead of N/A when no data
`fundamental_analysis()` in `financial_analytics.py` initialises `"financial_health_score": 0`. `_analyze_financial_health()` returns `None` when `metrics_found == 0` (which is correct). But the caller only sets `analysis["financial_health_score"]` when the return is not None — so the `0` default is never overwritten. DBS has no `currentRatio`/`quickRatio`/`debtToEquity` (banks don't use these) so health score stays `0.0/10`, which is factually wrong.

**Fix:** Change default to `None` and display `N/A` when null in the frontend.

**Verified yfinance fields available for D05.SI:**
```
revenueGrowth:              0.032   (3.2%)
earningsQuarterlyGrowth:    0.011   (1.1%)
earningsGrowth:             None    (annual, not provided for SGX)
currentRatio:               None    (bank, N/A)
quickRatio:                 None    (bank, N/A)
debtToEquity:               None    (bank, N/A)
```

## Tasks

### Task 1 — Map `revenueGrowth` in `yahoo_scraper.py`

File: `src/scrapers/yahoo_scraper.py` around line 197

After the existing `earningsGrowth` block, add:
```python
# Revenue Growth
if info.get("revenueGrowth") is not None:
    data["Revenue Growth (Yahoo)"] = f"{info.get('revenueGrowth') * 100:.2f}%"

# Earnings Quarterly Growth (fallback when annual earningsGrowth is absent)
if not info.get("earningsGrowth") and info.get("earningsQuarterlyGrowth") is not None:
    data["Earnings Growth (Yahoo)"] = (
        f"{info.get('earningsQuarterlyGrowth') * 100:.2f}%"
    )
```

Note: `_extract_metric()` uses partial match (`key.lower() in data_key.lower()`), so `'Revenue Growth'` matches `'Revenue Growth (Yahoo)'` and `'Earnings Growth'` matches `'Earnings Growth (Yahoo)'` — no changes needed in `financial_analytics.py` for growth.
