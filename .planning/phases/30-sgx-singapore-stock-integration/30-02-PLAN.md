# Phase 30-02 Plan: Chart NaN Fix for SGX Tickers

**Goal:** Price chart renders correctly for SGX tickers (e.g. `D05.SI`).

**Root cause:** `fetch_ohlcv` returns a trailing row for the current trading day where `Open/High/Low/Close` are `NaN` (intraday SGX data from yfinance). `df["Open"].tolist()` passes Python `float('nan')` values into `json.dumps`, which serialises them as the literal string `NaN` — invalid per JSON spec. The browser's `JSON.parse()` throws `SyntaxError`, triggering `_showError()` in `priceChart.js` → chart not rendered.

**Verified:**
```
NaN in Close: 1
NaN in Open: 1
json.dumps({"open": [1.0, float('nan')]}) → '{"open": [1.0, NaN]}'  # invalid JSON
```

---

## Tasks

### Task 1 — Drop NaN rows in `get_price_history` route (`webapp.py`)

File: `webapp.py` around line 2584

After `df = fetch_ohlcv(ticker, days)`, add:
```python
df = df.dropna(subset=["Open", "High", "Low", "Close"])
if df.empty:
    return jsonify({"error": "No price data available for this ticker"}), 200
```

This must happen before `dates = df.index.strftime(...)` so the index and all column lists stay aligned.

### Task 2 — Drop NaN rows in the main `/scrape` chart block (`webapp.py`)

File: `webapp.py` around line 2455

There is a second `fetch_ohlcv` call inside the main scrape handler (used for technical indicators / Plotly chart data assembled inline). Apply the same `dropna` before any `.tolist()` call on OHLCV columns.

Search pattern: `df = fetch_ohlcv(ticker, lookback)` / `df_365 = fetch_ohlcv(ticker, 365)` — add `dropna` after each.
