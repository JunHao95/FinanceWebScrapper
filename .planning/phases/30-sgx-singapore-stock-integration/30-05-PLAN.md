CHANGE 1 — Add short-name variant. After the existing block that appends resolved_name to search_terms (currently lines 130-131), insert the short_name extraction:

  search_terms = [ticker.lower()]
  resolved_name = company_name or company_names.get(ticker.upper())
  if resolved_name:
      search_terms.append(resolved_name.lower())
      short_name = resolved_name.split()[0].lower()
      if short_name not in search_terms:
          search_terms.append(short_name)
  if "." in ticker:
      base = ticker.rsplit(".", 1)[0].lower()
      if base not in search_terms:
          search_terms.append(base)

For D05.SI this produces: ['d05.si', 'dbs group holdings ltd', 'dbs', 'd05']
For AAPL this produces:   ['aapl', 'apple']  (split()[0] = 'apple', already in list, deduplicated)

CHANGE 2 — Fix the Google News URL (currently lines 142-144) to use only the company name, not the AND-join of all search_terms:

  BEFORE:
      "https://news.google.com/rss/search?q={}&hl=en-US&gl=US&ceid=US:en".format(
          "+".join(search_terms)
      ),

  AFTER:
      "https://news.google.com/rss/search?q={}&hl=en-US&gl=US&ceid=US:en".format(
          (resolved_name or ticker).replace(" ", "+")
      ),

Rationale: AND-joining all terms (including d05.si) creates a highly restrictive query that returns near-zero results. Using only the company name ("DBS+Group+Holdings+Ltd") lets Google News find articles naturally. The existing line-168 filter (any(term in text_to_search for term in search_terms)) then checks whether 'dbs' or 'dbs group holdings ltd' appears in each article — 'dbs' will match the majority.
