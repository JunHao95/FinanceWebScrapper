**Issue 1 — HEALTH-02 PARTIAL: Metric clicks navigate to tab top, not section**

`portfolioHealth.js` lines 171 and 180 call `TabManager.switchTab('analytics')` on VaR and Sharpe chip clicks. This switches to the Analytics tab but does not scroll to or anchor to the specific Sharpe/VaR subsection within the tab. `#analyticsResults` has no anchor IDs on individual subsections. The requirement says "links/jumps to its **relevant analytics tab section**" — section-level deep-link is not implemented.

**Issue 2 — AUTO-05 FRAGILE: Implicit globals from rlModels.js**

`autoRun.js` calls `rlEscapeHTML()` (4×) and `rlAlert()` (1×) as bare global references. These are plain `function` declarations inside `rlModels.js`, not exported via `window.*`. They work only because `rlModels.js` loads before `autoRun.js` via script tag order (`index.html:1952` before `1954`). No guard check exists in `autoRun.js`. If `rlModels.js` fails to load, MDP rendering throws `ReferenceError` silently. Neither function is in `REQUIRED_MODULES` in `main.js`.
